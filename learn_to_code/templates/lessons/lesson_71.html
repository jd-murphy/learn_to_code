Deep look at contains_prime()

Okay coder, let's look closely at the code from the last lesson. Remember, there is always more than one way to solve a programming problem! This is one possiblilty.

def contains_prime(num_list):
   for num in num_list:
     for i in range(2,num):
       if num % i == 0:
         break
     else:
       if num != 1:
         return f"Contains at least one prime! ({num})"
   return "No primes found in list."
1 / 9

We define the function with the def keyword. Inside the parentheses we provide a parameter called num_list to accept a list of numbers as an argument when we call the function.

2 / 9

On the second line of the function we use indentation to start the body of the function, which is the code that will run when we call the function. The first line of code in the body of the function is a for loop. We declare a variable num for each number in num_list. Each iteration through the loop, num will hold the current number from num_list.

3 / 9

Inside of the for loop, we code another for loop. This is a common technique called nesting. We can nest loops, if statements, and even functions. Nesting just means to write a loop within a loop or an if statement within an if statement. This second for loop allows us to do something amazing. For each number in num_list, we are going to create a range() from 2 up to (but not including) the current number from num_list. This means if the first number in num_list is 7 we will loop through the numbers 2, 3, 4, 5, and 6, one by one, and for each number, run the code in this nested for loop.

4 / 9

Once inside this inner for loop, we will be checking if num (the current number from num_list) is evenly divisible by i (the current number from the nested for loop).

5 / 9

If num % i == 0: is True, that means that num has i as a factor, which means num is not prime! As soon as we know that num is NOT prime we break out of the nested for loop, and flow of the program returns to the outer for loop, and we move on to the next number in num_list.

6 / 9

The else clause on the nested, inner for loop runs only if the inner, nested loop runs all the way to completion without a break statement being executed. This means no factors were found for num and it must be prime!

7 / 9

In the else clause for the inner, nested loop, we need to make sure we are not allowing the number 1 to be detected as prime. By math definition, the number 1 is not prime. So, if the current number from num_list (num) is not equal to 1 we know we have found a prime number. We looped through every possible number between 2 and the number itself and found no factors. Therefore, the number is prime.

8 / 9

At this point we return an f-string from the function. The returned string is formatted to display the prime number, so we know which prime was found.

9 / 9

If this doesn't happen for any number in num_list we know that no primes were found so we return a string saying no numbers in num_list are prime.
